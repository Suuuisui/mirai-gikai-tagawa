# みらい議会＠田川市 課題とロードマップ

2026-07-16 時点の棚卸し。運用手順は[運用保守ガイド](./20260716_1100_運用保守ガイド.md)を参照。

## 1. 既知の欠陥・リスク（重要度順）

### 🔴 高: 再シードのたびに全議案のURLが変わる

`packages/seed/tagawa/build-csv.ts` が毎回 `randomUUID()` で bill_id / diet_session_id を採番するため、
データ更新（全消し→再投入）のたびに `/bills/{id}` のURLがすべて変わる。

- 影響: 共有されたリンク・ブックマーク・検索エンジンのインデックスが全滅する。運用が続くほど被害が大きくなる
- 修正案: **決定的なUUID（uuid v5）に変更する**。名前空間UUID + `"{会期key}:{議案番号}:{提出者}"` から生成すれば、
  何度再生成しても同じIDになる。移行時に一度だけURLが変わるので、早いうちにやるべき

### 🔴 高: 再シードでAIインタビュー設定が消える

`clearAllData()`（`packages/seed/shared/helper.ts`）が `interview_configs` / `interview_questions` も削除するが、
CSVに再投入データが無い。現在は手動バックアップ→bill_id付け替え→復元で回避している（運用保守ガイド§4.1）。

- 修正案（いずれか）:
  1. seed対象から interview系テーブルを除外する（`TABLES_TO_CLEAR` から外す。ただし bill_id 外部キーが
     再シードで無効になるため、上記の決定的UUID化とセットでやると綺麗に解決する）
  2. 差分更新方式に変える（全消しをやめ、キー一致でupsert）

### 🟠 中: エラー時に空結果が永続キャッシュされる

web の `unstable_cache` はデータ取得エラーや0件を区別せずキャッシュする。DB障害・キー失効などの一時的な
エラーが「ずっと空のサイト」として固定化される。revalidateで復旧はできるが、気づくまで空のまま。

- 修正案: リポジトリ層で「エラー時はthrowしてキャッシュさせない」よう統一する／0件時の `revalidate` を短くする

### 🟠 中: VERCEL_TOKEN の定期失効でデプロイが止まる（対応待ち）

CLIログイン由来のトークンが短命で、これまでに4回デプロイが失敗している。
→ ダッシュボード発行の無期限トークンに差し替えれば恒久解決（運用保守ガイド§3）。**人間の作業待ち**。

### 🟡 低: 「もっと詳しく」（難易度hard）が未整備のまま機能だけ残っている

hard用の bill_contents を1件も用意していないため、トグルは非表示化し、2026-07-16に `getDifficultyLevel()` を
normal固定にした（hardのCookieが残ったブラウザで全ページ空表示になる不具合の修正）。

- `web/src/middleware.ts` は今も `?difficulty=hard` でCookieをセットするが、読む側が固定なので実害なし
- 方針を決める: hardコンテンツを整備して復活させる／機能ごと削除する

### 🟡 低: デプロイワークフローに品質ゲートがない

`.github/workflows/deploy.yml` は push されたコードをそのままデプロイする（lint/test/typecheck を実行しない）。
Vercel側のビルド失敗なら本番は無事だが、「ビルドは通るが壊れている」変更は素通りする。
push前のローカル検証（AGENTS.md）が唯一の砦。CIにlint+typecheck+testジョブを足すのが望ましい。

### 🟡 低: ローカルで rubyful 関連テストが落ちる

`web/src/lib/rubyful/*.test.ts` がローカル環境（localStorage周り）で16件失敗する。コード自体の欠陥ではなく
テスト環境の問題の可能性が高いが、「テストが常に赤い」状態は他の失敗を見逃す温床になるので直したい。

### 🟡 低: その他のデータ既知事項

- 令和7年12月定例会 議案第63号が未収録（出典ページで確認できなかった）
- 議員別の賛否データは市議会だより（広報紙PDF）にしかなく、未収録
- 新形式 `sb_secret_` キーが使えない（Supabase側の既知不具合。service_role JWTで回避中）

## 5. 検討して見送ったもの（再検討時のために理由を残す）

- **LiteRT.js（ブラウザ内ML推論）**: 本サービスのAI用途は日本語長文生成でサーバー側LLMが必須。
  オンデバイスモデルは品質・数百MBのダウンロードサイズの両面で市民向けサイトに不適。
  推論コストも痛点ではないため導入メリットがない（2026-07-16判断）
- **全611議案の構造化**: 会議録に実質情報がない議案(約350件)は構造化しても中身が薄まるだけなので対象外とした
- **suiii.co 配下へのホスティング**: 介護業務システム本番と同居になるため不可（アカウント運用ルール）
