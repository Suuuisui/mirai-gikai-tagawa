# トップページ「タグ別議案一覧」興味度スコア順ソート設計

## 目的

トップページの「タグ別議案一覧」（各タグ最大6件）は、これまで
`submitted_date`（議決日）の新しい順という機械的な並びだった。
その結果、補正予算や定型の条例改正など、閲覧して面白みの薄い議案が上位を占め、
否決・不認定・討論があった議案など「人間が読んで面白い・サイトの意義を感じる」
議案が下位に埋もれてしまっていた。

本設計では、DBスキーマやシードは変更せず、Webアプリ側（`web/`）の純粋関数として
「興味度スコア」を計算し、**トップページのタグ別セクションのみ**をスコア順に並べ替える。

## 対象範囲

- 対象: トップページの「タグ別議案一覧」（`get-bills-by-featured-tags.ts` 経由）のみ
- 対象外: タグ詳細ページ（「すべて見る」ページ、`get-bills-by-tag.ts` 経由）は
  従来通り `submitted_date` の新しい順のまま変更しない

## スコア表

`web/src/features/bills/shared/utils/interest-score.ts` の
`computeBillInterestScore()` が計算する。各ルールは独立に加点・減点され、合計値がスコアとなる。

| ルール | 条件 | 点数 | 理由 |
| --- | --- | --- | --- |
| 異例の議決 | `status_note` が `/否決\|不認定\|不採択\|修正議決\|懲罰\|戒告\|継続審議/` にマッチ | +50 | 議会が割れた・差し戻した案件はこのサイトの存在意義そのもの |
| 手動キュレーション済み | `is_featured === true` | +30 | 運営が「これは面白い」と選定済み |
| 票が割れた/討論があった | `content` に「反対討論」または「賛成多数」を含む | +25 | 賛否が割れた形跡がある |
| 論点解説あり | `content` に「## 議会での主な論点」を含む | +15 | AIが論点を書けるほど内容が深い |
| 当初予算 | `name` または `title` が「一般会計予算」を含み、かつ「補正」を含まない | +10 | 自治体の1年を左右する当初予算は補正予算より読み応えがある |
| 生活密着キーワード | `name+title+summary` の連結が `/給食\|学校\|子ど\|保育\|水道\|病院\|ごみ\|料金\|手数料\|図書館\|公園\|バス\|税/` にマッチ | +8 | 住民生活に直結するテーマは関心が高い |
| 説明資料PDFあり | `explanation_material_urls` が1件以上の配列 | +2 | 資料があると読み物として厚みが増す |
| 定型議案ペナルティ | `name` が `/任命\|選任\|同意を求める\|専決処分\|人権擁護委員\|固定資産評価/` にマッチ | -15 | 人事同意・専決処分の承認等は形式的で読み物として薄い |

## ソート順

`sortBillsTagRowsByInterestDesc()`（同ファイル）が以下の優先順位で並べ替える。

1. 興味度スコアの降順
2. `submitted_date` の降順（新しい順、`null` は末尾）
3. `id` の昇順（安定化のためのタイブレーク）

`bills` が `null` の行はスコア `-Infinity` 扱いとし、常に末尾に回す。

## 配線

- `web/src/features/bills/server/repositories/bill-repository.ts` の
  `findPublishedBillsByTag()` に第5引数 `order: "date" | "interest" = "date"` を追加。
  `"interest"` のときのみ `sortBillsTagRowsByInterestDesc` を使う。
  デフォルトは `"date"` のため、タグ詳細ページ等の既存呼び出しは変更不要。
- `web/src/features/bills/server/loaders/get-bills-by-featured-tags.ts` の
  `findPublishedBillsByTag` 呼び出しのみ `"interest"` を渡す。

## 調整方法

スコアの重み付けを調整したい場合は `interest-score.ts` 内の
`SCORE_*` 定数（named constant）を変更してデプロイし直すだけでよい。
DBマイグレーションやシードの変更は不要。

## テスト

`web/src/features/bills/shared/utils/interest-score.test.ts` に
各加点・減点ルールの単体ケースと、「否決議案が新しい定型議案より上に来る」
統合ケースを含むソートのテストを実装している。
