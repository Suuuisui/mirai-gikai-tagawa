## 3. 解決済みアーカイブ（旧版で課題だったもの）

### 🔴 再シードのたびに全議案のURLが変わる → 解決（2026-07-16）

決定的UUID（uuid v5）による採番に変更（`packages/seed/tagawa/uuidv5.ts`）。名前空間UUIDの変更は厳禁。
再シードしてもURLは変わらない。

### 🔴 再シードでAIインタビュー設定が消える → 解決（2026-07-16）

`seed:csv` が削除前に `interview_configs` / `interview_questions` を自動スナップショットし、投入後に
新しい bill_id へ付け替えて自動復元するよう変更（`packages/seed/csv/interview-restore.ts`）。本番で1件の
復元を実証済み。

### 🟠 エラー時に空結果が永続キャッシュされる → 解決（2026-07-16）

リポジトリ層で「エラー時はthrowしてキャッシュさせない」よう統一（PGRST116のみ正常系のnullとして扱う）。

### 🟠 VERCEL_TOKEN の定期失効でデプロイが止まる → 解決（2026-07-16）

ダッシュボード発行の無期限トークンに差し替え済み。

### 🟡 デプロイが品質チェックの結果を待たない → 解決（2026-07-16）

`deploy.yml` を `workflow_run` 方式に変更し、Code Check 成功後のみデプロイが走るようにした。

### 🟡 ローカルで rubyful 関連テストが落ちる → 解決（2026-07-17）

原因は Node v22+ の実験的 Web Storage API。`--localstorage-file` 未指定だとグローバル `localStorage` が
`undefined` を返す壊れた状態で存在し、vitest の jsdom environment が既存キーを上書きしない仕様のため
jsdom 本来の実装に置き換わらず21件が失敗していた（CIは Node 20 のため再現しない）。
`web/vitest.setup.ts` を新設し、localStorage が使用不能な場合のみメモリ実装で補うシムを追加して解決。

### 🟡 REVALIDATE_SECRET の本番値が手元にない → 解決（2026-07-17）

Vercel CLI から `vercel env pull --environment=production <出力先>` で本番の環境変数一式を
いつでも取得できることを確認し、実際に `/api/revalidate` へ POST して HTTP 200 を検証済み。
手順は[運用保守ガイド](./20260716_1100_運用保守ガイド.md)の§5に記載。

### 議員提出資料PDFの活用（フェーズ1） → 完了（2026-07-16）

全会期の議決結果ページからPDFリンクを収集し、611議案中569議案に紐付け。`bills.explanation_material_urls`
（jsonb）に格納し、議案詳細に「提出時の説明資料（PDF）」セクションを表示するようにした。

### 議員提出資料PDFの活用（フェーズ2） → 一部完了

PDF本文をテキスト抽出して解説文の情報源にする取り組み。r8-3/r8-4 の28件中、実質情報のある11件を
詳細解説化した。残り17件は同意書等で情報が薄く、意図的に見送り。過去会期への展開は低優先
（会議録ベースの解説が既にあるため）。

## 6. 検討して見送ったもの（再検討時のために理由を残す）

  オンデバイスモデルは品質・数百MBのダウンロードサイズの両面で市民向けサイトに不適。
